## Known Web/Blade Issues And Inconsistencies

- Candidate identity fields are currently on `users`, not `candidates`.
- Admin pendaftaran search may reference candidate columns that are currently user columns.
- `CandidateWebController::redirectCandidateUser()` may reference `interviewer.schedule`, while the route name is `interviewer.schedules`.
- Admin Profile Matching ranking calls are not read-only; they calculate/write `spk_results` and `spk_calculation_logs`.
- Candidate web documents page does not clearly handle `candidate_signature_path` and `parent_signature_path` even though those columns exist.
- Interviewer pendaftaran loads candidates broadly in current scan; department scoping is unclear.
